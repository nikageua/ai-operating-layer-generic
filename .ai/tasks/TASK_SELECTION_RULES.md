# Task Selection Rules

When selecting the next task, prefer:

1. the smallest task with the highest impact on the current slice
2. tasks aligned with `.ai/brain/NEXT_STEPS.md`
3. tasks that do not expand scope prematurely
4. smaller, testable tasks over larger architectural changes

Avoid selecting:
- large systems
- future roadmap items
- blocked tasks
- speculative work

If uncertain, propose the task but do not activate it without approval.
