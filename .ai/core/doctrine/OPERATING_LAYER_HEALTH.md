# Operating Layer Health

AIL v2 should be evaluated by whether it improves delivery quality, not by how tidy its files look.

## Signals that AIL is helping
- repeated rediscovery decreases
- shared conventions become easier to find
- fewer tasks fail twice for the same structural reason
- cross-repo surprises are caught earlier
- project docs influence future decisions instead of being ignored
- durable updates are infrequent but high-value

## Signals that overlay quality is good
- project docs stay small but useful
- profiles reduce guesswork instead of adding ceremony
- patterns and pitfalls actually get reused
- new sessions converge faster on correct context

## Signals that knowledge promotion is healthy
- promoted lessons are reused later
- candidate proposals are filtered, not auto-promoted
- durable docs stay higher-signal than runtime notes

## Signals of bureaucracy or drift
- many durable docs are rarely consulted
- updates are frequent but low-value
- story-local details keep leaking into project truth
- runtime notes pile up without cleanup
- teams create artifacts "because the system says so" rather than because coordination value exists

## Success question

Ask periodically:
"If we removed this document or rule, would future work become meaningfully worse?"
If the honest answer is no, that content is a candidate for pruning.
