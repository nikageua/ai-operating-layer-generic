# Adoption Path

AIL v2 should be adoptable gradually.
Teams should not need to fill the whole system on day one.

## Day 1 minimum
Start with:
- `PROJECT_CONTEXT.md`
- `REPO_REGISTRY.yaml`
- one or two project profiles if needed
- lanes, failure protocol, and naming rules understood

## First 3 to 5 stories
Add only what proves useful:
- story folders
- a few tasks
- local runtime mapping
- one or two promoted pitfalls or patterns if they genuinely repeat

## When to add stronger governance
Add stronger durable curation discipline when:
- multiple people or sessions work in parallel,
- repeated candidate learnings appear,
- project truth starts diverging,
- story-local lessons start competing for promotion.

## Rule

Adoption should follow pain.
Do not pre-install every governance mechanism before the team has earned the need for it.
