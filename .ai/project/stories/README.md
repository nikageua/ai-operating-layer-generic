# Stories

Use one folder per multi-repo or non-trivial story.

Recommended shape:

```text
STORY-012-user-invite-flow/
  STORY.md
  TASKS.md
  NOTES.md
```

A story is the coordination unit for work that is too broad for one task or spans multiple repos.
Use logical repo IDs in tracked story docs.
Do not copy the whole AIL per story.
