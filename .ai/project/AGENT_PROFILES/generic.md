# Profile: generic

## Applies to
- repo:<id>

## Use for
- small changes
- exploratory tasks
- tasks without a stronger project-specific profile

## Avoid when
- architecture is unclear
- the repo has strong conventions not yet captured
- a contract or boundary may change

## Validation
- run the smallest relevant validation for changed scope
- prefer local checks that match actual blast radius

## Escalate when
- the task spans multiple repos
- ownership is unclear
- the profile keeps forcing guesswork
