# Agent Profiles

Sub-agents that execute tasks are configured by profiles.
Each profile defines the harness for a specific target environment.

## What Is A Profile

A profile is the harness a sub-agent reads before starting a task. It includes:
- what framework and language are in use
- what tools and skills the agent has access to
- how to bootstrap the environment
- how to validate and test its own output
- known pitfalls for this stack
- AGENTS.md content expectations

## Available Profiles

| Profile ID | Framework / Stack | Status |
|------------|-------------------|--------|
| `generic` | Fallback for any stack | stable |
| `angular-frontend` | Angular (TypeScript) | draft |
| `dotnet-api` | .NET (C#) | draft |

Add a row here whenever a new profile is created.

## How Tasks Declare A Profile

Each task file contains:
```
## Agent Profile
[profile-id]
```

If the profile does not exist yet, the task must be preceded by a harness bootstrap
task (see `HARNESS_BOOTSTRAP.md`) that creates the profile.

## Profile Lifecycle

```
bootstrap → draft → validated → stable → deprecated
```

- **bootstrap**: auto-generated by the harness bootstrap protocol, not yet run
- **draft**: has been used in at least one task run, not yet optimized
- **validated**: has passed metaharness evaluation (tasks.json score ≥ threshold)
- **stable**: reliable enough to use without review
- **deprecated**: superseded by a newer profile
